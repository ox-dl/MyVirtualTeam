Troubleshooting

This page lists common issues that may occur when using MyVirtualTeam, along with recommended solutions and debugging tips.

Installation Issues

  • App doesn’t start after installation: Ensure .NET Desktop Runtime 8.0 or later is installed.
  • Windows blocked the installer: Click More info → Run anyway in the SmartScreen prompt.
  • Hyper-V not found: Enable Hyper-V in Windows features or check that your Windows edition supports it.

Hyper-V Errors

  • The virtual machine could not be started: Verify your user account has administrative privileges.
  • Network adapter not found: Create or select a valid Virtual Switch in Hyper-V Manager.
  • VMs not appearing in list: Refresh the dashboard or restart MyVirtualTeam as Administrator.

Projects Issues

  • Project fails to load: Check the project file for corruption or missing files.
  • VMs not created: Ensure sufficient disk space and permissions to create VHD files.
  • Import errors: Verify the source project is compatible and not corrupted.

Sequences Issues

  • Sequence fails to start: Check that all required VMs are running and accessible.
  • Actions not executing: Verify action configurations and network connectivity.
  • Sequence hangs: Review logs for errors and ensure no conflicting processes are running.

Performance and Resource Usage

  • VMs are slow to start: Allocate more CPU or RAM per VM.
  • Host is unresponsive: Reduce the number of concurrent virtual machines.
  • Disk fills quickly: Monitor VHD file growth and clean up old snapshots.

Reporting a Problem

If you can’t resolve your issue, please report it with:

  • Steps to reproduce
  • Your Windows and .NET version
  • Log file from %AppData%\MyVirtualTeam\Logs\
